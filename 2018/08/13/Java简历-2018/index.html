项目经验

[ 2018/02 - 至今 ] 物联网设备生命周期管理平台（SLC）

该产品是提供给北美电信运营商IOT设备管理的平台，整个产品基于Spring-Cloud技术栈的消息驱动微服务架构，主要使用了Zuul、Feign、Stream、Zookeeper。其中使用Kafka作为Stream底层接入消息队列；使用基于Zookeeper的一致性的分布式配置中心；使用Metrics+InfluxDB+Grafana方案作为指标收集、汇聚及展示平台。

主要负责Kafka服务搭建及使用API开发，Zookeeper配置中心搭建及使用，监控平台搭建及使用。

[ 2016/10 - 2018/02 ] 分布式数据访问中间件（DDAL）

该产品是重启Beehive项目后的演进版本，之前Beehive重点关注的是代理模式的数据库访问中间件，DDAL是集应用嵌入（实现JDBC驱动）及代理接入两种模式于一身的产品；在通用中间件解析、路由、SQL执行、结果集处理框架下加入了执行计划、实时监控等功能。

产品采用开源的方式进行管理，方便成员协作；同时运用了HikariCP光连接池，Metrics指标收集，InfluxDB时序数据库，grafana监控展示等工具。

主要参与产品架构选型，功能方案设计，模块代码开发；并编写单元测试，性能测试，集成测试，以及编写使用手册、功能详细设计文档。

[ 2016/03 - 2016/09 ] 途牛商旅系统（TTMS）

该系统基于基于分布式云计算架构设计理念，分为SAAS（应用层）、APAAS（业务服务层）、IPAAS（技术服务层）、IAAS（基础架构层）。各层应用服务提供的能力通过服务总线进行发现与编排；配置统一交由配置中心管理；各服务独立部署，具备分布式应用特性。

应用开发使用了SpringBoot框架，结合Springboot注解，大大降低了开发的复杂度，使开发者更专注于应用逻辑的实现。应用之间异步服务通过消息总线进行通信，保障了消息的可靠传输，也降低了系统之间的耦合，使各应用高度内聚。业务之间服务编排使用了Activiti流程引擎，轻量化，功能强大，具备分布式特性。

主要参与架构设计，业务服务梳理实现，Activiti流程嵌入开发，SAAS服务编写，整体集成等。

[ 2015/08 - 2016/02 ] 分布式消息中间件（USB）

该产品借鉴kafka轻量高性能的架构设计；是一款分布式，队列模型的消息中间件。具备严格保证消息顺序；消息无限堆积等能力；主从同步双写，数据多副本存储；读写分离，主从架构；GUI/CMD双运维工具。

基于Producer，Broker，Consumer三种角色的消息生产存储及消费模型。Producer生产消息并推送给Broker存储，Consumer把消息从Broker上提取出来（可采用Broker推送给Consumer模式）。其中Producer与Consumer以客户端组件嵌入用户代码，通过调用API实现消息的推送及拉取；Broker以独立的服务形式运行，接收到消息先写入内存，再实时刷盘写入文件。采用磁盘线性写入方式（线性追加）提升写入性能；利用Linux Page Cache充当文件系统与应用进程之间的缓冲池加速数据读性能；通过Zerocopy技术加速数据传输；具备按过期时间或磁盘空间占用率为策略的文件淘汰机制。GUI运维工具通过JMX管理接口，对各组建进行监控统计以及参数配置调整。

主要参与架构设计，详细程序设计，后台核心代码编写，系统测试与后期维护工作。主要包括Topic功能实现，消息存储模块，故障转移功能实现，运维工具开发。以及运用过程中生产者使用开发。

[ 2014/07 - 2015/07 ] 分布式数据访问中间件（Beehive）

该中间件是基于淘宝开源项目Cobar开发；用于MySQL读写分离和高可用，可实现表的水平垂直切分，底层支持异构数据库的分布式数据库访问中间件。

中间件整体架构基于Zookeeper（ZK）实现的高可用的故障自动切换。Agent模块读取各个数据库实例状态发送给ZK，Switch模块检测主句状态，剔除不可用主机形成切换方案。虚拟Schema应用读取Switch计算出的结果进行应用中连接数据主机的切换。

该平台支持数据源的读写分离和负载均衡；支持全局表，数据自动分片到多个节点，用于高效表关联查；支持多样性的数据分片策略，基于时间、前缀、子串、范围、取模等策略；支持跨数据节点的数据合并、排序、分页、sum\min\max\count；支持高性能的分布式全局自增sequence；多平台支持，部署和实施简单。

研发过程中主要参与架构设计；路由，缓存，序列模块的开发；文档编写及整理。

[ 2013/08 - 2014/06 ] 产品与套餐管理系统（PPM）

该项目采用Java + MongoDB开发，主要用于电信项目销售品管理。销售品结构非常复杂，因此重点在于前端填单，数据展示与统计，js的编写工作量比较大。前端样式用到了BootStrap及Matrix框架，扁平化风格。此外系统流程管理用到了JBPM工作流引擎，对于任务的分配及扭转能够很好的加以控制。

[ 2013/01 - 2013/05 ] 统一维护平台（UMP）

该项目基于Ruby + MongoDB开发，用于自定义数据库维护脚本运行平台。前台采用CoffeeScript编辑器,使得js的编写要容易的多，并且可读性也提高；样式使用的是BootStrap前端框架。其中前后台交互运用到了node.js，以实现前后台消息推送。项目代码是用Git进行管理的，具有灵活，分支易于管理等特点。代码部署发布使用的是Capistrano，实现自动化代码部署、代码回滚、命令执行。服务器端采用的是Nginx+Unicon架构，用于web服务及代理转发。总体来说，该项目采用敏捷开发模式，适合较小工程的快速推进。

[2012/010 - 2013/01 ] 任务管理系统

该项目基于Java + Mysql开发，用于内部任务的分配评分及绩效管理。前端采用EXT框架，模块化程度较高。前后台采用Servlet交互，便捷高效。项目提供了邮件服务以及EXCEL的导入导出等报表功能。
